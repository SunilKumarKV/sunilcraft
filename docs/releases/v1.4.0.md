# SunilCraft v1.4.0 — Multi-platform Coding Profile

## Summary

SunilCraft v1.4.0 extends the portfolio from a LeetCode-focused coding view into a broader multi-platform coding profile. This release adds Codeforces metadata support, platform summary handling, and cleaner navigation between the coding overview, problem tracker, codebase, and dashboard.

## Highlights

- Added a dedicated Coding hub experience for recruiter-friendly coding proof
- Added Codeforces metadata support across coding, problems, and dashboard views
- Added platform summary support for profile-level coding records such as CodeChef summaries
- Improved separation between Coding, Problems, Codebase, and Dashboard

## Features

- New `/coding` hub with:
  - profile overview
  - coding goal progress
  - platform breakdown
  - activity heatmap
  - recent accepted solves
  - learning timeline
- Codeforces metadata records now appear in:
  - coding totals
  - platform breakdowns
  - activity heatmap
  - recent solves
  - learning timeline
  - problem tracker entries
- Metadata-only problem detail pages now explain when solution code has not been attached yet
- Platform profile summaries are now shown separately from normal solved problems
- Navigation was simplified so Coding is the primary coding entry point, while Problems and Codebase remain available as deeper routes

## Fixes

- Normalized platform casing across coding data views
- Prevented metadata-only entries from appearing in the Codebase solution library
- Improved accepted-language handling by falling back to submission metadata when code files are missing
- Clarified page copy so Coding, Problems, Codebase, and Dashboard no longer sound interchangeable

## Build/Test Status

- `npm run build`: passed on the release branch
- `npm run build`: passed again on `main` after merge
- No additional automated frontend test suite is configured in this repo

## Known Limitations

- Some coding records are still metadata-only and do not yet have attached solution code
- The Codebase detail bundle remains the heaviest client-side chunk in the build
- Platform summary records provide solved totals, but they do not replace per-problem imported history
